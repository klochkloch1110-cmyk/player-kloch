# Быстрый старт — Запуск и тестирование AudioPlayer

## 🚀 Запуск проекта

### Предварительные требования

Убедитесь, что установлено:
- ✅ Node.js (v16+)
- ✅ npm или yarn
- ✅ React Native CLI
- ✅ Android Studio (для Android)
- ✅ Xcode (для iOS, только macOS)

### Шаг 1: Установка зависимостей

```bash
cd AudioPlayer
npm install --legacy-peer-deps
```

**Примечание:** Флаг `--legacy-peer-deps` нужен из-за конфликта React 19 с некоторыми библиотеками.

### Шаг 2: Подготовка Android (если еще не сделано)

#### Установка зависимостей Android
```bash
cd android
./gradlew clean
cd ..
```

#### Добавление разрешений (уже добавлены в AndroidManifest.xml)
Проверьте `android/app/src/main/AndroidManifest.xml`:
```xml
<uses-permission android:name="android.permission.READ_EXTERNAL_STORAGE" />
<uses-permission android:name="android.permission.READ_MEDIA_AUDIO" />
<uses-permission android:name="android.permission.FOREGROUND_SERVICE" />
<uses-permission android:name="android.permission.WAKE_LOCK" />
```

### Шаг 3: Подготовка iOS (только macOS)

```bash
cd ios
pod install
cd ..
```

### Шаг 4: Запуск Metro Bundler

Откройте **первый терминал** и запустите:
```bash
npm start
```

Или:
```bash
npx react-native start
```

Оставьте этот терминал открытым.

### Шаг 5: Запуск на устройстве/эмуляторе

#### Android

Откройте **второй терминал**:

**Вариант 1: Автоматический запуск**
```bash
npm run android
```

Или:
```bash
npx react-native run-android
```

**Вариант 2: Сборка и установка вручную**
```bash
cd android
./gradlew installDebug
cd ..
```

#### iOS (только macOS)

```bash
npm run ios
```

Или:
```bash
npx react-native run-ios
```

**Для конкретного симулятора:**
```bash
npx react-native run-ios --simulator="iPhone 15"
```

### Шаг 6: Добавление тестовых файлов

#### На Android:

1. **Подключите устройство через USB** или используйте эмулятор
2. **Скопируйте MP3 файлы** в папку Music:

```bash
# Через adb
adb push /path/to/your/music.mp3 /sdcard/Music/

# Или используйте Android File Transfer (macOS)
# Или просто скопируйте файлы через проводник (Windows)
```

3. **Рекомендуется** использовать файлы с:
   - ID3v2 тегами (для проверки метаданных)
   - Embedded обложками
   - Кириллицей в названиях

#### На iOS:

1. Используйте **iTunes** или **Finder** (macOS Catalina+)
2. Синхронизируйте музыкальные файлы с симулятором/устройством

**Или добавьте файлы через Xcode:**
1. Откройте `ios/AudioPlayer.xcworkspace`
2. Добавьте файлы в папку ресурсов

## 🧪 Что можно протестировать сейчас

### 1. Базовый UI и навигация ✅

**Что проверить:**
- Запустите приложение
- Проверьте переключение между вкладками: Library, Albums, Artists, Playlists
- Нажмите на иконку эквалайзера в правом верхнем углу
- Проверьте работу темной/светлой темы (может быть в настройках позже)

**Ожидаемый результат:**
- Плавные переходы между экранами
- Neumorphic дизайн (тени, рельеф)
- Все UI элементы отзывчивы

### 2. Сканирование медиатеки ✅

**Как тестировать:**
1. Откройте вкладку **Library**
2. Приложение запросит разрешения:
   - На Android 13+: "Allow AudioPlayer to access music?"
   - На Android <13: "Allow AudioPlayer to access files?"
3. Нажмите **Allow** / **Разрешить**
4. Нажмите кнопку **"Scan Library"** (если треки не загрузились автоматически)
5. Дождитесь завершения сканирования

**Что проверить:**
- [ ] Все MP3/M4A файлы найдены
- [ ] Названия треков корректны (из ID3, не из имени файла)
- [ ] Исполнители отображаются правильно
- [ ] Альбомы определены
- [ ] Обложки появляются (если есть embedded)

**Возможные проблемы:**
- Если файлы не найдены → проверьте путь к Music директории
- Если пустой список → убедитесь, что файлы скопированы

### 3. Воспроизведение аудио ✅

**Как тестировать:**
1. Тапните на любой трек в списке
2. Внизу появится **мини-плеер**
3. Тапните на мини-плеер → откроется **NowPlayingScreen**

**Что проверить:**
- [ ] Аудио начинает воспроизводиться
- [ ] Обложка отображается (если есть)
- [ ] Название и исполнитель корректны
- [ ] Прогресс-бар движется
- [ ] Время воспроизведения обновляется
- [ ] Кнопка Play/Pause переключается
- [ ] Кнопки Next/Previous работают

### 4. Фоновое воспроизведение ✅

**Как тестировать:**
1. Начните воспроизведение трека
2. Сверните приложение (Home button)
3. Музыка продолжает играть
4. Проверьте notification panel:
   - Должно быть уведомление с контролами
   - Название трека и исполнитель
   - Кнопки Play/Pause/Next/Previous

**Что проверить:**
- [ ] Музыка не останавливается при сворачивании
- [ ] Уведомление отображается
- [ ] Контролы в уведомлении работают
- [ ] Lock screen показывает плеер (iOS/Android)

### 5. ID3 Метаданные и обложки ✨ НОВОЕ

**Как тестировать:**

**Подготовьте тестовые файлы:**
- `test1.mp3` — с ID3v2 тегами и обложкой
- `test2.mp3` — только ID3v1 (без обложки)
- `no_tags.mp3` — без ID3 тегов (fallback тест)
- `russian.mp3` — с кириллицей в тегах

**Что проверить:**

✅ **Метаданные читаются:**
```javascript
// Откройте Chrome DevTools для проверки логов
// В Metro терминале будут видны console.log
```

- [ ] Файлы с ID3v2 показывают правильный title/artist/album
- [ ] Файлы с ID3v1 работают (ограниченная информация)
- [ ] Файлы без тегов показывают имя файла как title
- [ ] Кириллица отображается корректно

✅ **Обложки отображаются:**
- [ ] В списке треков (TrackList)
- [ ] В мини-плеере (MiniPlayer)
- [ ] На экране воспроизведения (NowPlayingScreen)
- [ ] Качество изображений хорошее
- [ ] Нет артефактов или растягивания

✅ **Дополнительные поля:**
- [ ] Год (year) — проверьте в store/логах
- [ ] Жанр (genre)
- [ ] Номер трека (trackNumber)

### 6. Эквалайзер ✅

**Как тестировать:**
1. Начните воспроизведение
2. Нажмите иконку эквалайзера (правый верхний угол)
3. Переключите эквалайзер в ON
4. Выберите пресет (Pop, Rock, Jazz, Classical, Flat)
5. Или настройте вручную слайдерами

**Что проверить:**
- [ ] Переключатель ON/OFF работает
- [ ] Пресеты переключаются плавно
- [ ] Слайдеры двигаются
- [ ] UI отзывчив

**Примечание:** На данный момент эквалайзер **UI-only** (визуальный, без реального DSP). Реальная обработка звука будет добавлена позже.

### 7. Контролы плеера ✅

**Что проверить:**

## 🐛 Отладка проблем

### Проблема: "Metro bundler not running"

**Решение:**
```bash
npm start -- --reset-cache
```

### Проблема: "Unable to load script"

**Решение:**
```bash
# Остановите Metro
# Очистите кэш
npm start -- --reset-cache

# В другом терминале
npm run android
```

### Проблема: Файлы не найдены при сканировании

**Android:**
```bash
# Проверьте, что файлы скопированы
adb shell ls /sdcard/Music/

# Проверьте разрешения
adb shell pm list permissions -g
```

**Решение:**
- Убедитесь, что разрешения даны
- Проверьте правильность путей в `mediaLibrary.ts`
- Перезапустите приложение после копирования файлов

### Проблема: TypeScript ошибки

```bash
npx tsc --noEmit
```

Если есть ошибки — сообщите, исправлю.

### Проблема: Обложки не отображаются

**Проверьте в логах Metro:**
```
[MetadataService] Success: Song Title Artist Name
[MetadataService] Cover art extracted: data:image/jpeg;base64,...
```

Если лога нет → файл без обложки или ошибка чтения.

### Проблема: Build failed

**Android:**
```bash
cd android
./gradlew clean
cd ..
npm run android
```

**iOS:**
```bash
cd ios
pod deintegrate
pod install
cd ..
npm run ios
```

```bash

cd android && ./gradlew clean && cd ..
```
