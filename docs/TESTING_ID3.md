# Инструкция по тестированию ID3 метаданных

## Подготовка

1. Убедитесь, что зависимости установлены:
```bash
npm install --legacy-peer-deps
```

2. Проверьте TypeScript компиляцию:
```bash
npx tsc --noEmit
```

3. Соберите проект:
```bash
# Android
npx react-native run-android

# iOS
npx react-native run-ios
```

## Тестовые сценарии

### 1. Базовое чтение метаданных

**Цель:** Проверить, что метаданные читаются из ID3 тегов

**Шаги:**
1. Откройте экран "Library" (Библиотека)
2. Нажмите "Scan Library" (если медиатека пустая)
3. Дождитесь завершения сканирования

**Ожидаемый результат:**
- Треки отображаются с правильными названиями
- Исполнители извлекаются из ID3, а не из имени файла
- Альбомы корректно определены

**Что проверить:**
- [ ] Треки с ID3 тегами отображают правильный title/artist/album
- [ ] Треки без ID3 тегов используют fallback на имя файла
- [ ] Нет ошибок в консоли

### 2. Обложки треков

**Цель:** Проверить отображение embedded обложек

**Шаги:**
1. Найдите трек с embedded обложкой в медиатеке
2. Тапните по треку для воспроизведения
3. Откройте NowPlayingScreen (тап на мини-плеер)

**Ожидаемый результат:**
- Обложка отображается в мини-плеере
- Обложка отображается на экране воспроизведения
- Изображение не искажено

**Что проверить:**
- [ ] Обложки JPEG отображаются корректно
- [ ] Обложки PNG отображаются корректно
- [ ] Треки без обложек показывают placeholder
- [ ] Нет утечек памяти при переключении треков

### 3. Дополнительные метаданные

**Цель:** Проверить извлечение year, genre, trackNumber

**Шаги:**
1. Используйте файл MP3 с полными метаданными
2. Добавьте `console.log` в `mediaLibrary.ts` после `createTrack()`
3. Запустите сканирование

**Что проверить:**
- [ ] `year` извлекается и парсится как число
- [ ] `genre` читается корректно
- [ ] `trackNumber` парсится из формата "3/12"
- [ ] Опциональные поля `undefined` для файлов без тегов

### 4. Производительность

**Цель:** Проверить скорость сканирования с ID3

**Шаги:**
1. Поместите 50+ MP3 файлов в Music директорию
2. Запустите сканирование
3. Измерьте время

**Ожидаемый результат:**
- Сканирование завершается за разумное время
- UI не зависает
- Прогресс заметен

**Что проверить:**
- [ ] 50 файлов сканируются менее чем за 30 секунд
- [ ] 100+ файлов сканируются менее чем за минуту
- [ ] Нет блокировки UI во время сканирования

### 5. Обработка ошибок

**Цель:** Проверить fallback при ошибках чтения

**Тестовые файлы:**
- MP3 без ID3 тегов
- Поврежденный MP3 файл
- Файл с неполными метаданными

**Что проверить:**
- [ ] Файл без ID3 использует имя файла для title
- [ ] Поврежденный файл не ломает сканирование
- [ ] Частичные метаданные заполняют только доступные поля

### 6. Различные форматы

**Цель:** Проверить поддержку разных аудио-форматов

**Тестовые файлы:**
- MP3 с ID3v2.4
- MP3 с ID3v1
- M4A с iTunes tags
- FLAC с Vorbis Comments

**Что проверить:**
- [ ] MP3 ID3v2.4 читается полностью
- [ ] MP3 ID3v1 читается с ограничениями
- [ ] M4A теги извлекаются
- [ ] FLAC теги извлекаются

### 7. Кириллица

**Цель:** Проверить поддержку кириллицы в метаданных

**Шаги:**
1. Используйте MP3 с русскими названиями в ID3
2. Запустите сканирование

**Ожидаемый результат:**
- Кириллица отображается корректно
- Нет кракозябр или mojibake

**Что проверить:**
- [ ] Русские названия треков отображаются правильно
- [ ] Русские имена исполнителей корректны
- [ ] Русские названия альбомов читаются

## Известные ограничения

1. **Duration:** jsmediatags не извлекает длительность
   - Решение: используется react-native-track-player

2. **Большие обложки:** могут замедлять сканирование
   - Рекомендация: кэшировать обложки

3. **Экзотические форматы:** OGG, APE не поддерживаются
   - Планируется: расширение поддержки
